# api/v1

This directory contains API specifications and contracts.

## Purpose
- Define API schemas and contracts
- Document API endpoints
- OpenAPI/Swagger specifications
- Protocol buffer definitions (if using gRPC)
- API versioning

## What goes here
- OpenAPI/Swagger YAML files
- API documentation
- Request/response schema definitions
- API versioning documentation
- Example requests/responses

## Why version your API?
Starting with v1 allows you to evolve your API over time while maintaining backward compatibility.
