# internal/config

This directory contains configuration management.

## Purpose
- Load and parse configuration from various sources
- Provide typed configuration structures
- Validate configuration values
- Support environment variables, config files, flags, etc.

## What goes here
- Configuration struct definitions
- Config loading/parsing logic
- Default values
- Configuration validation
- Environment variable mapping

## Example configurations
- Server port and host
- Worker pool size
- Timeout values
- Rate limiting parameters
- Log levels
